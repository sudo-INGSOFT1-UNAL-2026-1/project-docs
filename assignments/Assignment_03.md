## APARTADO 2 – Compromisos del grupo
 
- Se utilizará un **tablero Kanban** (por ejemplo, GitHub Projects) para organizar las tareas del proyecto. Las actividades se clasificarán en estados como *pendiente*, *en progreso* y *terminado*, y cada integrante será responsable de mantener actualizadas sus tareas.

- Se realizarán **reuniones de seguimiento** al menos **una vez por semana**, de forma presencial o virtual, para revisar avances, resolver dudas y ajustar la distribución de tareas cuando sea necesario.

- En cuanto al uso de **Git y GitHub**, el grupo acordó:
  - Trabajar mediante **ramas individuales** para cada aporte o funcionalidad.
  - Evitar realizar cambios directos sobre la rama `main`.
  - Revisar los cambios antes de hacer merge, ya sea mediante pull request o revisión grupal.
  - Mantener mensajes de commit claros y relacionados con el cambio realizado.

- Cada integrante se compromete a **avisar con anticipación** al grupo cuando no pueda cumplir con una tarea o con una fecha acordada, con el fin de reorganizar el trabajo y evitar retrasos.

- Las **decisiones técnicas y organizativas** se tomarán de manera conjunta, buscando el consenso del grupo. En caso de desacuerdos, se elegirá la opción más viable según el tiempo disponible y los objetivos del proyecto.
